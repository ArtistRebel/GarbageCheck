Improve the GarbageCheck app by adding multi-photo estimation for the same waste pile (naval).

Important:
- The user interface language is Russian.
- All new and modified UI text must be in Russian.
- In the UI, replace the word "куча" with the more professional term "навал".
- Keep code structure and internal variable names unchanged unless necessary.
- Do not rewrite the application.
- Do not change the existing volume calculation algorithm.
- Do not break the current single-photo workflow.

Current workflow:
Фото → контур навала → высота навала → масштаб → плотность → расчёт объёма

Goal:
Allow users to upload several photos of the same навал from different angles to improve estimation accuracy.

New user flow:

1. User uploads the first photo.
2. User draws the contour of the навал.
3. User enters height, scale and density.
4. App calculates the estimated volume.
5. User can choose:
"Добавить фото этого же навала"

6. User uploads another view of the same навал.
7. User repeats the contour and measurements if needed.
8. App calculates each photo independently.
9. App combines the results into one final estimate.
